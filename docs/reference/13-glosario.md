# Glosario de términos

**Audiencia:** Todos los usuarios  
**Propósito:** Definiciones claras de los conceptos clave utilizados en la plataforma.

---

| Término | Definición |
|---------|------------|
| **Asignación** | Relación entre un trabajo académico y un evaluador encargado de revisarlo. |
| **Cartel** | Modalidad de presentación visual (póster) donde el estudiante expone su trabajo en un formato gráfico. |
| **Evaluación en vivo** | Evaluación que realiza el jurado durante la presentación oral o de cartel del estudiante (fase 2). |
| **Evaluador** | Miembro del comité académico encargado de revisar y calificar los trabajos. |
| **Fase 1** | Primera etapa de evaluación: revisión de resúmenes. Los evaluadores califican los trabajos por su contenido escrito. |
| **Fase 2** | Segunda etapa de evaluación: evaluación de presentaciones orales o carteles durante el evento. |
| **Ganador** | Trabajo que obtiene el puntaje más alto en su categoría (oral o cartel) y es seleccionado por el administrador. |
| **Google Apps Script (GAS)** | Plataforma de Google que permite extender y automatizar Google Workspace. Es el backend del sistema. |
| **Google Sheets** | Hoja de cálculo de Google utilizada como base de datos del sistema. |
| **Google Drive** | Servicio de almacenamiento en la nube donde se guardan los PDFs de los trabajos y los certificados. |
| **Modalidad** | Forma de presentación del trabajo: oral (exposición con diapositivas) o cartel (póster). |
| **Oral** | Modalidad de presentación donde el estudiante expone su trabajo verbalmente, apoyado con diapositivas. |
| **Panel de administración** | Interfaz para que los administradores gestionen usuarios, trabajos, evaluaciones y configuración del evento. |
| **Panel de estudiante** | Interfaz donde los estudiantes envían trabajos y consultan su estado y retroalimentación. |
| **Panel de evaluador** | Interfaz donde los evaluadores revisan trabajos asignados, realizan evaluaciones y consultan su agenda. |
| **Progressive Web App (PWA)** | Aplicación web que puede instalarse en el dispositivo y funciona offline parcialmente. |
| **Resumen** | Texto breve (máx. 250 palabras) que describe el trabajo académico. |
| **Rúbrica** | Conjunto de criterios de evaluación con puntajes parciales. Ej: claridad, contenido, relevancia, originalidad. |
| **Service Worker** | Script que se ejecuta en segundo plano en el navegador, permitiendo funcionalidad offline y caché. |
| **Sesión** | Token de autenticación que se almacena en `localStorage` para mantener la sesión del usuario activa. |
| **Token** | Cadena alfanumérica única que identifica una sesión de usuario o un proceso de recuperación de contraseña. |
| **Trabajo** | Documento académico enviado por un estudiante para su evaluación en el encuentro. |
| **Web App** | Tipo de despliegue de Google Apps Script que expone el backend como un endpoint HTTP. |
